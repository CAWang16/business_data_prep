# Online Retail II — Data Processing & Analysis

**CSP 571 · Spring 2026 · Carl & Anders**

End-to-end pipeline for ingesting, validating, and analysing the
[UCI Online Retail II dataset](https://archive.ics.uci.edu/dataset/502/online+retail+ii)
(1,067,371 transactions, Dec 2009 – Dec 2011).

---

## Quick start

### 1. Prerequisites

- Python 3.9 or later
- `git`

### 2. Clone the repository

```bash
git clone <repo-url>
cd carl
```

### 3. Create a virtual environment and install dependencies

```bash
python3 -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate

pip install -r requirement.txt
```

### 4. Run the pipeline

All scripts must be run **from the project root** (the folder containing `data/`, `scripts/`, etc.).

#### Step 1 — Ingest the raw data into SQLite

```bash
python scripts/setup_db.py
```

Reads `data/raw/online_retail_II.xlsx` (two sheets, ~44 MB), combines them, and writes
`database/retail.db` (~99 MB). This takes roughly 25–30 seconds.

#### Step 2 — Validate raw field metadata

```bash
python scripts/00_validate_source_metadata.py
```

Prints a structured validation report for every column: format conformance,
missing-value counts, value-range checks, and anomaly summaries (A-prefix invoices,
non-standard StockCodes, etc.).

#### Step 3 — Split-decision EDA

```bash
python scripts/01_split_decision_eda.py
```

Cross-tabulates transaction types (sales, cancellations, free items, admin adjustments,
gift vouchers, DCGS codes) and documents the seven-category split logic used in
downstream cleaning.

#### Step 4 — Generate the report

```bash
# Word document (.docx)
python scripts/generate_report.py

# LaTeX source for Overleaf (.tex)
python scripts/generate_report_tex.py
```

Both scripts read directly from `database/retail.db` and write output files to the
project root: `report.docx` and `report.tex`.

## Using the LaTeX report on Overleaf

1. Upload `report.tex` to a new Overleaf project.
2. Upload the `figures/` folder (all three `.png` files) into the same project.
3. Compile with **pdfLaTeX** (default compiler — no changes needed).

The `figures/` path is referenced relatively, so the folder structure must mirror the
local layout.

## Dataset notes

| Property | Value |
|---|---|
| Source | [UCI ML Repository — Online Retail II](https://archive.ics.uci.edu/dataset/502/online+retail+ii) |
| Rows | 1,067,371 (combined sheets) |
| Fields | InvoiceNo, StockCode, Description, Quantity, InvoiceDate, UnitPrice, CustomerID, Country |
| Date range | 01 Dec 2009 – 09 Dec 2011 |

The dataset is included in `data/raw/` for reproducibility.
`database/retail.db` is excluded from git (see `.gitignore`) and must be regenerated
locally via `setup_db.py`.

## Troubleshooting

**`unable to open database file`** — Run `setup_db.py` from the project root, not from
inside `scripts/`. The script writes to `database/retail.db` relative to the working
directory.

**`ModuleNotFoundError`** — Make sure the virtual environment is activated
(`source venv/bin/activate`) before running any script.

**`openpyxl` warnings about large files** — Expected. The xlsx is ~44 MB with 1 M+ rows;
loading takes ~25 s.
